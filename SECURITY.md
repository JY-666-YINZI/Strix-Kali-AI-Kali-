# Security Policy

## Supported Versions

We actively maintain and provide security patches for the following deployment flavors:

| Version | Supported |
| ------- | --------- |
| 0.1.x   | ✅ Yes     |
| < 0.1.0 | ❌ No      |

## Reporting a Vulnerability

**Do not open public GitHub issues for security vulnerabilities.** 

If you discover a security flaw or an unexpected privilege escalation vector inside the Strix execution sandbox, please report it responsibly:
1. Email a detailed write-up to `security@strix-ops.local` (or your personal email).
2. Include steps to reproduce, the active LLM context configuration used, and the underlying OS state.
3. We will acknowledge your report within 48 hours and coordinate a patch within 7 days.

## Legal Disclaimer & Ethical Use

Strix-Kali-AI is strictly designed for authorized penetration testing, internal infrastructure auditing, and educational DevSecOps research. 
* The developers and maintainers accept **zero liability** for any damage, service disruption, or data loss caused by misuse of this tool.
* Launching attacks against unauthorized targets is a criminal offense globally. **Test only what you own or have explicit written consent to audit.**
