# 支持指南 / Support Guide

欢迎来到 Strix-Kali-AI 用户支持与技术交流生态！在寻求人工支持之前，建议您先查阅以下官方资源，了解您的疑问是否已被解决。

## 🔍 自助排查资源

* **沙箱拉取卡死问题**：如果您的智能体在启动时卡在 `0/32 layers`，请参阅 [README 的沙箱优化章节](README.md#3-accelerated-sandbox-setup-domestic-mirror-fix)，使用国内镜像并进行本地 Docker 标签（Tag）重构。
* **Token 消耗与成本控制**：在开启高脑力模式（`STRIX_REASONING_EFFORT="high"`）前，请务必查阅我们文档中的“模型矩阵”，以防止因模型多轮深度推理引发非预期的 Token 额度快速消耗。

## 💬 交流与反馈渠道

### 1. GitHub Issues（问题反馈）
* **Bug 提交**：若您遇到依赖库崩溃、CLI 参数解析错误或系统级异常，请使用我们标准的 Issue 模版提交报告。
* **新需求提案**：如果您希望工具支持新的大模型服务商（例如：月之暗面 Moonshot、百川智能等），欢迎提交带有 `feature-request` 标签的提案。

### 2. GitHub Discussions（社区讨论）
如果您想参与工具架构的讨论、分享您独创的 `--instruction` 战术小纸条配置，或者展示您通过 AI 成功挖掘并验证的零误报 PoC 案例，欢迎前往本仓库的 **Discussions** 板块与全球安全白帽子一同交流。
